# 盲道检测系统接口设计规范

## 📋 概述

本文档定义了盲道检测系统中各模块间的通信接口规范，包括RESTful API、WebSocket实时通信、数据格式标准化和错误处理机制。

## 🏗️ 系统架构

### 模块组成
- **检测模块** (Detection Module): 负责盲道和障碍物检测
- **Android应用模块** (Android Module): 移动端数据采集和用户交互
- **语音导航模块** (Voice Module): 语音提示和导航指导
- **主控制器** (Main Controller): 系统协调和模块管理

### 通信模式
- **同步通信**: RESTful API用于配置、状态查询等
- **异步通信**: WebSocket用于实时数据传输
- **消息队列**: 用于模块间解耦和可靠性保证

## 🌐 RESTful API 规范

### 基础信息
- **基础URL**: `http://localhost:8080/api/v1`
- **认证方式**: Bearer Token
- **数据格式**: JSON
- **字符编码**: UTF-8

### 通用响应格式

#### 成功响应
```json
{
  "success": true,
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": "2024-01-01T12:00:00Z"
}
```

#### 错误响应
```json
{
  "success": false,
  "code": 400,
  "message": "请求参数错误",
  "error": {
    "type": "ValidationError",
    "details": "具体错误信息"
  },
```

## 🔌 WebSocket 实时通信

### 连接信息
- **URL**: `ws://localhost:8080/ws`
- **协议**: 盲道检测系统专用协议
- **心跳间隔**: 30秒

### 消息格式
```json
{
  "type": "message_type",
  "id": "unique_message_id",
  "timestamp": "2024-01-01T12:00:00Z",
  "sender": "module_name",
  "receiver": "module_name|broadcast",
  "data": {}
}
```

### 消息类型

#### 1. 检测结果推送
```json
{
  "type": "detection_result",
  "id": "msg_001",
  "sender": "detection_module",

  "data": {
```


## ⚠️ 错误处理机制

### HTTP状态码规范

| 状态码 | 含义 | 使用场景 |
|--------|------|----------|
| 200 | 成功 | 请求成功处理 |
| 201 | 创建成功 | 资源创建成功 |
| 400 | 请求错误 | 参数验证失败 |
| 401 | 未授权 | 认证失败 |
| 403 | 禁止访问 | 权限不足 |
| 404 | 未找到 | 资源不存在 |
| 409 | 冲突 | 资源冲突 |
| 422 | 参数错误 | 业务逻辑错误 |
| 429 | 请求过多 | 限流触发 |
| 500 | 服务器错误 | 内部错误 |
| 503 | 服务不可用 | 服务维护 |

### 错误类型定义

#### 验证错误 (ValidationError)
```json
{
  "success": false,
  "code": 400,
  "message": "请求参数验证失败",
  "error": {
    "type": "ValidationError",
    "details": [
      {
        "field": "image_data",
        "message": "图像数据不能为空",
        "code": "REQUIRED"
      },
      {
        "field": "confidence_threshold",
        "message": "置信度阈值必须在0-1之间",
        "code": "RANGE_ERROR"
      }
    ]
  }
}
```

#### 业务逻辑错误 (BusinessError)
```json
{
  "success": false,
  "code": 422,
  "message": "检测服务暂时不可用",
  "error": {
    "type": "BusinessError",
    "details": "模型正在更新中，请稍后重试",
    "retry_after": 300
  }
}
```

#### 系统错误 (SystemError)
```json
{
  "success": false,
  "code": 500,
  "message": "内部服务器错误",
  "error": {
    "type": "SystemError",
    "details": "数据库连接失败",
    "error_id": "err_123456",
    "support_contact": "support@blindroad.com"
  }
}
```

## 🔒 安全规范

### 认证机制
- **JWT Token**: 用于API认证
- **API Key**: 用于服务间通信
- **OAuth 2.0**: 用于第三方集成

### 数据加密
- **传输加密**: TLS 1.3
- **存储加密**: AES-256
- **敏感数据**: 字段级加密

### 访问控制
- **RBAC**: 基于角色的访问控制
- **API限流**: 防止滥用
- **IP白名单**: 限制访问来源
